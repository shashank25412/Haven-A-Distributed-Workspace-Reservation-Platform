14. Coding standards:
(Rules we'll enforce throughout the project.)
	- Prefer composition over inheritance.
	- Use std::unique_ptr by default; std::shared_ptr only with clear shared ownership.
	- Avoid raw new/delete.
	- Make value objects immutable.
	- Favor constructor injection.
	- Avoid singleton patterns except for stateless framework-managed components if truly necessary.
	- Use std::optional instead of nullable pointers where appropriate.
	- Keep exceptions for exceptional conditions, not control flow.
	- Follow RAII consistently.
